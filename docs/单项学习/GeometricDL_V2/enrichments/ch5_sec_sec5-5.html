🔍 深入理解：什么是 E(3) 等变性？

❓ 小白：$E(3)$ 是什么群？为什么分子/粒子系统需要 E(3) 等变性？和置换不变性有什么区别？

💡 专家：$E(3)$ = 3D 欧几里得群，包含所有保持距离不变的刚体变换。

$E(3)$ 的组成：

  • 平移（Translations）：$\mathbf{x} \mapsto \mathbf{x} + \mathbf{t}$，$\mathbf{t} \in \mathbb{R}^3$
  • 旋转（Rotations）：$\mathbf{x} \mapsto R\mathbf{x}$，$R \in SO(3)$（旋转矩阵，$R^\top R = I$, $\det R = 1$）
  • 组合：$\mathbf{x} \mapsto R\mathbf{x} + \mathbf{t}$

为什么物理系统需要 E(3) 等变性？

  • 平移不变性：物理定律在任何位置都相同（牛顿定律不依赖绝对坐标）
  • 旋转不变性：空间没有"绝对方向"（在地球上还是在太空中，力学规律相同）
  • 例子：分子的能量只依赖原子间相对位置，与分子整体放在哪里、如何旋转无关

置换 vs E(3) 等变性：

对称性作用对象含义例子
置换不变性
$\Sigma_n$		 节点编号 重新排列粒子编号不影响结果 粒子 1,2,3 = 粒子 3,1,2
E(3) 等变性 3D 空间坐标 旋转/平移整个系统，输出也旋转/平移 旋转分子 → 预测的力也旋转

标准 GNN 的问题：

  • GCN/GAT/GraphSAGE 只保证置换不变性
  • 但它们不保证 E(3) 等变性！
  • 如果直接把 3D 坐标 $(x,y,z)$ 作为特征输入 GNN，旋转分子后，输出会完全不同

反例（标准 GNN 失败）：

  • 输入：氢分子 H₂，原子坐标 $[x_1=(0,0,0), x_2=(1,0,0)]$
  • GNN 预测：原子 1 受力 $\mathbf{F}_1 = (1, 0, 0)$
  • 旋转分子 90°：$x_1'=(0,0,0), x_2'=(0,1,0)$
  • 如果 GNN 是 E(3) 等变的，应该预测 $\mathbf{F}_1' = (0, 1, 0)$
  • 但标准 GNN 会预测错误的方向（因为它只是学习了数据模式，没有理解旋转）

E(3) 等变的严格定义：

对任意 $R \in SO(3)$, $\mathbf{t} \in \mathbb{R}^3$，如果函数 $F$ 满足：

$$F(R\mathbf{x}_1 + \mathbf{t}, \ldots, R\mathbf{x}_n + \mathbf{t}) = R \cdot F(\mathbf{x}_1, \ldots, \mathbf{x}_n) + \mathbf{t}$$

则称 $F$ 是 E(3) 等变的。

生活类比：

  • 想象你在测量两根棍子之间的夹角
  • 旋转整张桌子（空间旋转）→ 夹角不变（E(3) 不变性）
  • 交换两根棍子的标签（置换）→ 夹角不变（置换不变性）
  • 物理量（如力、速度）需要同时满足这两种对称性

🔍 深入理解：EGNN 的核心公式逐步推导

❓ 小白：EGNN 的公式看起来很复杂：
$f_u' = \phi(f_u, \bigoplus_v \psi_f(f_u, f_v, \|x_u - x_v\|^2))$
$x_u' = x_u + \sum_v (x_u - x_v) \psi_c(f_u, f_v, \|x_u - x_v\|^2)$
为什么这样设计就能保证 E(3) 等变性？

💡 专家：EGNN 的天才之处：用最简单的设计达到 E(3) 等变性。逐步理解：

Step 1：区分标量特征和坐标

  • $f_u$：标量特征（如原子类型、质量）— 旋转/平移不变
  • $x_u \in \mathbb{R}^3$：3D 坐标 — 旋转/平移下会变化
  • 关键思想：分别处理两类信息

Step 2：标量特征更新（E(3) 不变）

$$f_u' = \phi\left(f_u, \bigoplus_{v \in \mathcal{N}_u} \psi_f(f_u, f_v, \|x_u - x_v\|^2)\right)$$
  • $\|x_u - x_v\|^2$：距离的平方（E(3) 不变量）
  • $\psi_f$：消息函数，输入是标量（$f_u, f_v$）和距离（不变量）
  • 输出 $f_u'$ 只依赖不变量 → 旋转/平移输入时，$f_u'$ 不变 ✅

Step 3：坐标更新（E(3) 等变）

$$x_u' = x_u + \sum_{v \neq u} (x_u - x_v) \psi_c(f_u, f_v, \|x_u - x_v\|^2)$$
  • $\psi_c$：输出标量权重（不变）
  • $(x_u - x_v)$：向量（相对位移）
  • 关键：标量 × 向量 = 新向量

Step 4：等变性证明（旋转）

设旋转变换 $\mathbf{x} \mapsto R\mathbf{x}$，我们要证明 $x_u' \mapsto Rx_u'$。

  • 原始更新：$x_u' = x_u + \sum_v (x_u - x_v) w_v$（简记 $w_v = \psi_c(\cdots)$）
  • 旋转后输入：$\tilde{x}_u = Rx_u$，$\tilde{x}_v = Rx_v$
  • 距离不变：$\|\tilde{x}_u - \tilde{x}_v\|^2 = \|R(x_u - x_v)\|^2 = \|x_u - x_v\|^2$
  • 因此权重不变：$\tilde{w}_v = \psi_c(f_u, f_v, \|\tilde{x}_u - \tilde{x}_v\|^2) = w_v$
  • 旋转后的更新： $$\begin{align*} \tilde{x}_u' &= \tilde{x}_u + \sum_v (\tilde{x}_u - \tilde{x}_v) \tilde{w}_v \\ &= Rx_u + \sum_v (Rx_u - Rx_v) w_v \\ &= Rx_u + R\sum_v (x_u - x_v) w_v \\ &= R\left(x_u + \sum_v (x_u - x_v) w_v\right) \\ &= R x_u' \end{align*}$$
  • 证毕 ✅

Step 5：平移等变性（更简单）

  • 平移 $\mathbf{x} \mapsto \mathbf{x} + \mathbf{t}$
  • 相对位移：$(x_u + \mathbf{t}) - (x_v + \mathbf{t}) = x_u - x_v$（平移抵消）
  • 因此 $x_u' \mapsto x_u' + \mathbf{t}$ ✅

关键洞察：

  1. 只用相对量：$x_u - x_v$（不用绝对坐标 $x_u$）→ 平移不变
  2. 只用不变量：$\|x_u - x_v\|$（不用向量本身）→ 旋转不变的标量
  3. 标量 × 向量：$w (x_u - x_v)$（标量权重乘以向量）→ 旋转等变

为什么不直接用 $(x_u - x_v)$ 作为特征？

  • 如果 $\psi_f$ 接收向量 $(x_u - x_v)$，旋转后向量会变 → $f_u'$ 会变 → 不再是不变量
  • 只用距离 $\|x_u - x_v\|$：丢弃了方向信息，只保留长度 → 完全不变

🔍 深入理解：Tensor Field Networks 与不可约表示

❓ 小白：EGNN 只能处理标量特征，如果节点特征本身是向量（如速度）或张量（如应力），怎么办？

💡 专家：Tensor Field Networks (TFN) 和 SE(3)-Transformer 使用不可约表示理论，可以处理任意类型的张量。

问题背景：

  • EGNN 假设所有特征是标量（0 阶张量）
  • 但物理中很多量是向量（1 阶张量）或矩阵（2 阶张量）：
  • 例子：力 $\mathbf{F}$、速度 $\mathbf{v}$、角动量 $\mathbf{L}$（向量）；应力张量 $\boldsymbol{\sigma}$（矩阵）

核心数学：不可约表示（Irreducible Representations, Irreps）

  • 旋转群 $SO(3)$ 的任何表示都可以分解为不可约块（就像矩阵的特征值分解）
  • 标准基：球谐函数 $Y_l^m(\theta, \phi)$，其中 $l$ 是阶数（order）
  • $l=0$：标量（1 个分量）
  • $l=1$：向量（3 个分量，对应 $m = -1, 0, +1$）
  • $l=2$：2 阶张量（5 个独立分量，对应无迹对称张量）

如何表示向量？

  • 笛卡尔坐标 $(x, y, z)$ 不是不可约表示（旋转时三个分量会混合）
  • 球坐标 $(Y_1^{-1}, Y_1^0, Y_1^{+1})$ 是不可约表示： $$Y_1^{-1} = \frac{x - iy}{\sqrt{2}}, \quad Y_1^0 = z, \quad Y_1^{+1} = -\frac{x + iy}{\sqrt{2}}$$
  • 旋转 $R$ 后，每个 $Y_1^m$ 只与同阶的 $Y_1^{m'}$ 线性组合（$3 \times 3$ Wigner D-矩阵）

Tensor Field Network 的核心思想：

  1. 将所有特征表示为不可约表示的直和：$f = \bigoplus_{l=0}^{L_{\max}} f^{(l)}$
  2. $f^{(0)}$：标量部分（如原子类型）
  3. $f^{(1)}$：向量部分（如速度）
  4. $f^{(2)}$：2 阶张量部分
  5. 消息传递时，用Clebsch-Gordan 系数（CG 系数）组合不同阶的特征
  6. CG 系数保证组合后仍然是等变的

CG 系数的作用：

  • 问题：如何从两个向量 $f^{(1)}$ 和 $g^{(1)}$ 生成一个新的标量/向量/张量？
  • 答案：用 CG 系数 $C_{l_1 m_1, l_2 m_2}^{l m}$ 组合
  • 例子（向量 × 向量）：
    • → 标量（$l=0$）：$f \cdot g$（点积）
    • → 向量（$l=1$）：$f \times g$（叉积）
    • → 2 阶张量（$l=2$）：对称无迹部分 $f \otimes g$

为什么这么复杂？

  • 代价：数学复杂（需要球谐函数、CG 系数）
  • 收益：理论保证的等变性 + 可以处理任意物理量
  • 适用场景：分子动力学、量子化学、天体物理

现代简化方法：E(n)-GNN (Satorras et al., 2021)

  • 用 EGNN 的思路：不显式表示旋转，只用不变量 + 等变构造
  • 速度特征：用速度的模 $\|\mathbf{v}\|$ 作为标量，方向信息隐式包含在坐标更新中
  • 更简单但表达力略低（无法显式处理角动量等高阶量）

🔍 深入理解：PhysRobot 的 EdgeFrame 设计

❓ 小白：项目代码里的 EdgeFrame 是怎么编码粒子间关系的？为什么要分别存储 $\mathbf{r}_{ij}$ 和 $\|\mathbf{r}_{ij}\|$？

💡 专家：EdgeFrame 的设计直接来自 EGNN 理论，但针对动力学系统优化。

完整特征列表（8 维）：

edge_features = [
    r_x, r_y, r_z,        # 相对位移（向量，3D）
    ||r||,                 # 距离（标量，不变量）
    v_x, v_y, v_z,        # 相对速度（向量，3D）
    ||v||                  # 速率（标量，不变量）
]

为什么需要向量 + 标量？

特征类型E(3) 性质物理意义
$\mathbf{r}_{ij}$向量等变粒子 $j$ 相对 $i$ 的方向
$\|\mathbf{r}_{ij}\|$标量不变粒子间距离（如弹簧长度）
$\mathbf{v}_{\text{rel}}$向量等变相对运动方向（碰撞/分离）
$\|\mathbf{v}_{\text{rel}}\|$标量不变相对速率（碰撞强度）

设计原理：

  1. 向量特征（$\mathbf{r}, \mathbf{v}$）：提供方向信息
    • 用于计算力的方向（如弹簧力沿 $\mathbf{r}$ 方向）
    • 必须保留完整的 3D 信息
  2. 标量特征（$\|\mathbf{r}\|, \|\mathbf{v}\|$）：提供幅度信息
    • 用于计算力的大小（如胡克定律 $F = k \|\mathbf{r} - \mathbf{r}_0\|$）
    • E(3) 不变 → 可以直接输入 MLP

具体应用（弹簧力）：

  • 物理公式：$\mathbf{F}_{ij} = -k (\|\mathbf{r}_{ij}\| - l_0) \frac{\mathbf{r}_{ij}}{\|\mathbf{r}_{ij}\|}$
  • 分解：
    • $k (\|\mathbf{r}_{ij}\| - l_0)$：标量权重（取决于距离）
    • $\frac{\mathbf{r}_{ij}}{\|\mathbf{r}_{ij}\|}$：单位向量（方向）
  • EdgeFrame 提供：$\|\mathbf{r}_{ij}\|$ 和 $\mathbf{r}_{ij}$ → MLP 学习权重 $w$，然后 $\mathbf{F} = w \cdot \mathbf{r}_{ij}$

反对称性验证：

  • 牛顿第三定律：$\mathbf{F}_{ij} = -\mathbf{F}_{ji}$
  • EdgeFrame 保证：$\mathbf{r}_{ij} = -\mathbf{r}_{ji}$（相对位移反向）
  • 如果消息函数是对称的（$\psi(f_i, f_j, e_{ij}) = \psi(f_j, f_i, e_{ji})$），加权后自然满足反对称性

代码实现要点：

# 相对位移
r_ij = positions[tgt] - positions[src]  # (E, 3)
r_norm = torch.norm(r_ij, dim=1, keepdim=True)  # (E, 1)

# 相对速度
v_rel = velocities[tgt] - velocities[src]  # (E, 3)
v_norm = torch.norm(v_rel, dim=1, keepdim=True)  # (E, 1)

# 拼接：[向量, 标量, 向量, 标量] = 8 维
edge_feat = torch.cat([r_ij, r_norm, v_rel, v_norm], dim=1)

扩展：加入角度信息（未来改进）

  • 当前 EdgeFrame 是成对的（只看两个粒子）
  • 三体作用（如弯曲力）需要角度：$\angle(i, j, k)$
  • 可以扩展为 3-body EdgeFrame：包含两个相对向量和它们的夹角

🎯 直觉理解：等变性 = "正确旋转"

生活类比：指南针

  • 你有一个指南针，它指向北方
  • 旋转你自己（空间旋转）→ 指南针的指针也相对你旋转（但仍指向北）
  • 这就是等变性：输出"跟随"输入旋转
  • 如果指南针是不变的（总是指向同一方向），那就是"不变性"

物理直觉：

  • 标量（如温度、质量）：旋转/平移空间，值不变 → 不变
  • 向量（如力、速度）：旋转空间，方向跟着旋转 → 等变
  • 张量（如应力）：旋转空间，所有分量按特定规则变换 → 等变

为什么 EGNN 的设计"优雅"？

  • 不需要显式处理旋转矩阵 $R$（避免复杂的球谐函数）
  • 只用相对量和不变量→ 自然满足等变性
  • 简单、高效、可解释

🏥 医疗机器人应用：E(3) 等变性的实际价值

场景 1：手术机器人的力感知

  • 问题：机器人手臂接触软组织，需要预测接触力
  • 挑战：接触点的空间方向随机，不能假设"总是从上往下接触"
  • 解决：E(3) 等变网络 → 无论手臂从哪个角度接触，预测的力方向都正确
  • 数据效率：只需要一个方向的训练数据，自动泛化到所有方向

场景 2：PhysRobot 的粒子仿真

  • 当前实现：EdgeFrame（相对位移 + 距离）→ 天然 E(3) 等变
  • 好处：无论患者体位如何（仰卧/侧卧/俯卧），仿真结果物理一致
  • 扩展：加入速度 EdgeFrame（已实现）→ 处理动态碰撞

场景 3：分子对接（药物设计）

  • 问题：预测药物分子与蛋白质的结合姿态
  • 挑战：分子可以从任意角度接近蛋白质
  • 方法：E(3) 等变 GNN（如 SchNet, DimeNet）预测结合能
  • 优势：旋转不变性 = 不需要枚举所有可能的对接角度

❓ 小白：这么多 E(3) 等变方法，应该用哪个？

💡 专家：选择指南：

方法复杂度特征类型表达力速度适用场景
EGNN 简单 标量 中 快 粒子系统、点云、分子动力学
SchNet 中等 标量 + 径向基 中 快 量子化学（能量预测）
TFN 复杂 任意张量 高 慢 需要角动量、应力张量的系统
SE(3)-Trans 复杂 任意张量 + 注意力 最高 慢 蛋白质折叠、复杂分子
PhysRobot
DynamicalGNN		 简单 标量 + EdgeFrame 中 快 实时软组织仿真

决策树：

  • 只需要标量特征（位置、类型）？→ EGNN
  • 需要预测标量能量（量子化学）？→ SchNet
  • 需要向量/张量特征（力、应力）？→ TFN 或 SE(3)-Transformer
  • 实时性要求高？→ EGNN（最快）
  • 表达力要求最高（不在乎速度）？→ SE(3)-Transformer

PhysRobot 为什么选择 EGNN 思路？

  1. 实时性：医疗机器人需要 30-60 Hz 更新（16-33ms 每帧）
  2. 简单性：EdgeFrame 容易理解和调试
  3. 物理先验：牛顿力学只需要标量力的大小 + 方向向量
  4. 可扩展：未来可以加入角动量（升级到 TFN）
